Guia rápido

Orientações relativas à recolha de dados no domínio das Pescas e da Comunidade

Um guia rápido para as estruturas e protocolos de recolha de dados do inquérito KoboToolBox.

Introdução

O Guia Rápido fornece orientações passo-a-passo para recolher e validar dados sobre a pesca e a comunidade através do sistema. Esta orientação resume o processo de recolha de dados e descreve os protocolos de actualização dos ficheiros de referência e dos quadros de controlo de qualidade.

O tribunal desenvolveu quatro formulários de inquérito para recolher dados sobre os desembarques de pesca e as comunidades piscatórias. As descrições detalhadas da pesquisa estão disponíveis no Guia Completo em Inglês, Bahasa Indonésio, Cebuano, Francês, Tagalo, Suaíli, Malgaxe, Espanhol, Português e Tetun.

O Inquérito de monitorização dos desembarques capta os dados mínimos, idealmente diários ou sempre que possível, necessários para acompanhar as tendências a longo prazo da produção pesqueira, da composição das espécies e do valor das capturas. Este inquérito pode ajudar a avaliar os efeitos dos encerramentos das pescas ou de outras estratégias de gestão, comparando dados antes e depois da execução. O inquérito é curto e fácil de utilizar para permitir uma recolha de dados mais frequente e coerente.

O Inquérito de perfis de desembarques recolhe dados pormenorizados sobre as artes de pesca, o esforço e a dimensão de peixes ou invertebrados individuais, juntamente com informações especializadas para grupos específicos de espécies, tais como sexo, qualidade das capturas, dados sobre estiletes e maturidade dos ovos. A qualidade da captura do polvo inclui métodos de armazenamento, Cor, Número de tentáculos, abrasões físicas, qualidade do bico e condições da pele. Os dados do estilete de polvo incluem fotos de amostras e circunferência de crescimento. Os dados de maturidade dos óvulos são responsáveis pelo peso das gónadas, cor, estádio, comprimento e aparecimento de linhas nas glândulas ovarianas dos polvos fêmeas. Estes dados complementam os dados de monitorização dos desembarques e ajudam os parceiros a compreender melhor as tendências das capturas observadas na produção pesqueira e os efeitos da Gestão das pescas. O inquérito foi concebido para ser realizado no início do envolvimento numa comunidade e, posteriormente, realizado de um a dois anos, antes e depois da aplicação da estratégia de gestão, ou conforme necessário para explicar as alterações das capturas observadas através de dados de monitorização.

O inquérito comunitário sobre o perfil reúne uma compreensão abrangente do panorama socioeconómico e das pescas de uma comunidade. O inquérito é utilizado quando se envolve pela primeira vez com uma comunidade e repetido quando apropriado para documentar grandes mudanças no comportamento da pesca, nas percepções dos Pescadores e na gestão dos recursos. A frequência do inquérito dependerá do âmbito do projecto (ou seja, quando se espera que ocorra uma alteração).

O Inquérito às famílias de pescadores capta informações pormenorizadas sobre as famílias de pescadores, tais como a perceção a nível das famílias das actividades relacionadas com a pesca, as fontes de rendimento, as actividades diárias, As principais espécies de peixes capturadas, a dependência do peixe para alimentação e a actividade de pesca. A pesquisa é usada quando se envolve pela primeira vez com uma comunidade e repetida a cada dois a cinco anos ou após intervenções de gestão para documentar mudanças no comportamento da pesca, percepções dos Pescadores, eficácia da Gestão, Segurança Alimentar, confiança e equidade de gênero.

Passos para utilizar formulários KoboToolbox

  1. Preparar dados de referência
    1. Os dados de referência podem ser atualizados através do Smartsheet e contêm informações abrangentes sobre os níveis administrativos das comunidades piscatórias (País, Província, Distrito/município, aldeia/comunidade), locais de desembarque, pescadores, compradores, coletores de dados, dados de espécies, áreas de gestão, tipos de habitat, tipos de artes e organizações parceiras.
    2. As folhas de dados de referência são utilizadas para preencher listas suspensas nos formulários de inquérito do KoboToolBox e juntar dados adicionais necessários para a análise, como as coordenadas da aldeia e o género do pescador.
    3. Actualizar os dados de referência com novas espécies, pescadores, compradores, locais de desembarque, aldeias, etc. que são necessários para a recolha de dados em novas áreas.
    4. É fundamental que estes ficheiros estejam actualizados para que a recolha e a análise dos dados funcionem sem problemas. Os ficheiros de referência são enviados para a kobo a cada 12-24 horas.
  1. Configurar a conta KoboToolbox
    1. Todos os inquéritos são administrados através da utilização de um formulário KoboToolBox. Cada um dos formulários de inquérito pode ser partilhado com uma conta KoboToolBox.
    2. Configure a sua conta Kobo Toolbox no servidor geral em https://kf.kobotoolbox.org/accounts/signup/
    3. Contacte a equipa com os dados da sua conta Kobo e o seu nome de utilizador para que os formulários apropriados possam ser partilhados com a sua conta.

  1. Descarregue a aplicação KoboCollect.

    1. Os formulários Kobo podem ser acedidos através da aplicação móvel KoboCollect free para Android ou através do formulário Web Enketo.

  2. Carregar formulários na aplicação KoboCollect. Consulte a configuração da Kobo para obter orientações adicionais.

    1. Depois de transferir a aplicação, configure o URL do servidor, o nome de Utilizador e a palavra-passe, que liga a aplicação KoboCollect ao servidor KoboToolbox. Isso permite que você baixe formulários implantados do KoboToolbox para o seu dispositivo móvel e também envie os dados coletados através do aplicativo de volta ao servidor. Para a URL, use a URL do servidor de uso geral - https://kc.kobotoolbox.org.
    2. Você pode acessar “configurações do projeto” clicando no ícone no canto superior direito da tela inicial.
    3. Na tela inicial, selecione ” Baixar formulário.”
    4. Uma lista de todos os seus formulários de pesquisa implantados deve aparecer. Pressione “Selecionar tudo” para que todos os formulários de pesquisa sejam enviados para o aplicativo ou selecione os que deseja, selecionando-os manualmente. Em seguida, clique em ” obter selecionado.”
    5. É importante que os inquéritos sejam actualizados regularmente para garantir que a versão mais recente do formulário está a ser utilizada de modo a não causar problemas no pipeline de dados. Embora o formulário Web seja atualizado automaticamente quando uma nova versão do questionário estiver disponível, os usuários devem configurar a “atualização automática” no aplicativo. Você pode configurar a atualização automática acessando “Configurações”e” gerenciamento de formulários”.
  1. Recolha de dados com os formulários de inquérito KoboToolBox
    1. Recolha dados seleccionando” Iniciar novo formulário ” na aplicação KoboCollect. Ver instruções pormenorizadas sobre a recolha de dados utilizando a aplicação na configuração Kobo.
    2. No final da pesquisa, selecione “Finalizar.”Se você quiser editar o formulário antes de enviar, Selecione “Salvar como rascunho.”Se você selecionou ” Salvar como rascunho”, seu formulário será enviado para ” rascunhos.”Aqui, você pode fazer edições em formulários antes de enviá-los para o conjunto de dados final. Depois de fazer alterações, selecione “Finalizar”, conforme mencionado acima. Depois de selecionar “Finalizar”, o formulário é automaticamente armazenado em ” Pronto para enviar.”Na versão atual do Kobo, você ainda pode editar formulários aqui. No futuro, no entanto, você não poderá editar uma vez finalizados, e “pronto para enviar” será apenas um espaço de armazenamento para armazenar formulários finalizados até que você tenha uma conexão com a internet para enviá-los ao servidor. Selecione “Enviar selecionado” para enviar todos os formulários para o servidor.
    3. Os dados recolhidos através da aplicação ou do formulário web serão limpos e armazenados num sistema central de armazenamento de dados. Os dados brutos são armazenados em cópia de segurança na Base de dados KoboToolbox.
  1. Validar dados no Smartsheet
    1. Todos os dados que fluem através do sistema de dados das pescas são validados no Smartsheet. Um guia de instruções passo a passo completo para concluir este processo encontra-se na página de Validação de dados.
    2. Os dados são enviados para validação em dois cenários: 1) novos nomes para um pescador, comprador, coletor de dados ou espécie são adicionados manualmente no Kobo Collect, ou 2) um valor numérico ultrapassa os limiares estabelecidos. Essas entradas devem ser verificadas antes de serem incluídas no conjunto de dados final, visualizadas no painel e adicionadas aos arquivos de referência.
    3. Quaisquer novos nomes ou dados sinalizados são enviados para a tabela de controle de qualidade relevante no Smartsheet para que as equipes de campo validem. As tabelas de controlo de qualidade são específicas do parceiro e podem ser acedidas através de pedidos de atualização enviados para o seu e-mail ou visitando as ligações de visualização dinâmica que lhe são fornecidas. Os líderes da equipa de campo serão notificados por e-mail quando os dados precisarem de ser validados nas tabelas de controlo de qualidade.
    4. Validação de novos nomes: quando um novo nome é introduzido durante a recolha de dados, estas entradas são automaticamente cruzadas com os ficheiros de referência para avaliar se já podem existir novas entradas nos ficheiros de referência, mas talvez escritas de forma diferente. Quaisquer entradas existentes semelhantes serão adicionadas à tabela sob as colunas “ref_match”. Os líderes de equipa podem então verificar facilmente se os “novos” dados já estão, de facto, nos ficheiros de referência. Os líderes de equipe completarão a coluna” validation_options “em cada tabela selecionando new_data, a entrada de dados de referência cruzada do arquivo de referência (ou seja, um” ref_match”), outro ou remover. Se estiver seleccionado” outro”, deve indicar um nome na coluna” outro”. Se for um novo nome de espécie local que precise ser validado, uma coluna com a foto tirada durante a coleta de dados será fornecida para referência.
    5. Validando valores numéricos com avisos de limite: para quaisquer entradas de dados sinalizadas, os líderes de equipe concluirão a validação de dados na coluna “correto” selecionando Y (Sim, os valores estão corretos), N (não, os valores não estão corretos) ou remover (os dados não podem ser validados e devem ser removidos do conjunto de dados mestre). Se não (‘N’) for seleccionado, os valores correctos para o numerador ou denominador devem ser introduzidos nas colunas” correct_numerator “e” correct_denominator”.
    6. Uma vez verificados os dados as tabelas de validação, acrescentaremos novas informações ao ficheiro de referência e forneceremos as correcções necessárias ao conjunto de dados principal.

  1. Atualizar folha de traduções

    1. As traduções para todos os idiomas são concluídas editando as folhas apropriadas no Smartsheet.

  2. Visualizar e utilizar dados

    1. Análises e visualizações automatizadas estarão disponíveis através de painéis padronizados. Os conjuntos de dados também estarão disponíveis por meio de download no painel ou link direto para outro banco de dados.

Por favor, deixe-nos saber se você tiver alguma dúvida.
